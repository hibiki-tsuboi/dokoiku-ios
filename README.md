# アプリ名

- どこ行く

# 概要

- ユーザーが「行きたい場所」や「食べたいお店」をあらかじめ登録しておき、迷ったときにアプリが今日の行き先を決めてくれるiPhoneアプリです。
- このアプリの核は、単なるランダム抽選ではありません。ユーザーが登録して忘れていた候補を再発掘し、「あ、ここ最近行ってないから行ってみたい！」と思える体験を作ることです。
- キャッチコピー「迷ったら、行き先をおまかせ。」
- 体験価値
    - 行きたい場所やお店をストックできる
    - 迷ったときに、今ちょうどよさそうな候補を出してくれる
    - 最近行っていない場所を思い出させてくれる
    - ごはん・おでかけのどちらにも使える
    - 決める瞬間が少し楽しい

# 機能

1. 候補の登録
2. 候補の一覧表示
3. 候補の編集・削除
4. 「ごはん」「おでかけ」「どちらでも」のモード選択
5. モードに合う候補からおすすめを出す
6. おすすめは **メイン1件 + サブ候補2件** を表示する
7. おすすめ理由を表示する
8. 「ここにする」ボタンで最後に行った日を今日に更新する
9. 「もう一回」ボタンで再レコメンドする
10. 空の状態ではサンプル候補または登録導線を表示する

# データモデル

- 候補の登録画面で下記の項目を登録できること
    - 名前
    - カテゴリ
        - ごはん / おでかけ / どちらでも
    - エリア
        - 渋谷、横浜、千葉など
    - メモ
    - 行きたい度
    - 最後に行った日
    - 値段
        - 安い、普通、高い
    - 行った回数

# モード設計

- ホーム画面で、ユーザーに以下の3つを選ばせてください。

1. **ごはん**
2. **おでかけ**
3. **どちらでも**

## ごはん

夕方や夜に使うことが多い。
飲食店・カフェ・ラーメン・焼肉などを優先。

## おでかけ

休日や朝から使うことが多い。
観光地・公園・買い物・散歩・イベント・日帰りスポットなどを優先。

## どちらでも

とにかく何かしたいとき。
ごはんとおでかけの両方から候補を出す。

## 基本方針

おすすめ度は以下を考慮してください。

1. 行きたい度が高い
2. 最近行っていない
3. 選択モードに合っている
4. まだ一度も行っていない
5. 毎回同じ候補にならないよう、少しランダム性を入れる
6. レコメンドは完全ランダムではなく、スコアリング + 少しのランダム性で実装してください。
